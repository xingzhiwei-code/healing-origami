## 〇、给所有新会话 AI 看（含我重启后的自己）

在你做**任何代码或决策**之前，**必须**按下面顺序完成「破冰」三件套：

### 1. 必读文档（按顺序读完，不要跳）

| 序 | 文档 | 用途 |
|---|---|---|
| 1 | `.cursorrules` | 工程宪法（最高优先级，与任何文档冲突以本文件为准） |
| 2 | `specs/ui-design.md` | 视觉 Design Tokens 唯一来源 |
| 3 | `specs/game-logic.md` | 玩法规范 |
| 4 | `specs/system-arch.md` | 系统架构规范 |
| 5 | `specs/PROGRESS.md` | 实时进度活文档 |

### 2. 破冰回答模板（新会话第一次响应必须严格按这个格式）

```
当前进度：<一句话，从 PROGRESS.md「当前状态」抄出>
正在做的里程碑：<M? · 名称>
上一步停在：<最近一项已完成或进行中的子任务>
建议下一步：<继续 X / 切换到 Y / 修复 Z>
是否按这个推进？
```

**禁止**：跳过破冰直接动代码、动文件、改架构、给 PRD。

### 3. 推进每一步完成后，立刻做的事

- 更新 `specs/PROGRESS.md`：勾掉刚完成项、追加决策记录、更新「下一步」。
- 如新增事件：先注册到 `assets/scripts/utils/EventManager.ts` 的 `GameEvents`。
- 如新增 Token：同步改 `specs/ui-design.md` 与 `assets/scripts/ui/DesignTokens.ts`（M5 后）。
- 如新增已决策项：追加到本文档 §二「已决策」。

## 一、AI 协作纪律（额外约束，`.cursorrules` 之上的补强）

- **中文优先**：用户为中文母语者，所有沟通与注释中文。
- **不主动创建 `.meta` 文件**：交给 Cocos Creator 编辑器生成。
- **不修改自动生成目录**：`temp/` `library/` `profiles/` `settings/` 一律不动。
- **注释只解释「为什么」**：禁止 narrative 注释（如 `// 增加计数`）。
- **不瞎猜 Cocos 3.8 API**：不确定时用 Context7 MCP 查最新文档，关键字 `cocos creator 3.8`。
- **不擅自升级引擎或依赖大版本**：需与维护者确认。
- **新代码只产出业务必需的最小集合**：禁止「顺手」加文档 / README / 测试，除非用户明确要求。
- **与 `.cursorrules` 冲突时**：以 `.cursorrules` 为准；本文档仅在 `.cursorrules` 未覆盖处起补强作用。

这些是已经经过用户与 AI 共同讨论敲定的事。新会话**不要**重新质疑这些决策，除非用户主动要求重新评估。

| # | 决策 | 简要原因 |
|---|---|---|
| D1 | 引擎：Cocos Creator 3.8.8 | 已对比 LayaAir / Unity / 纯 Web；Cocos 微信小游戏支持最完善 + 编辑器加速关卡制作 |

| D5 | 无计时 / 无星级 / 无失败惩罚 | 与「治愈」基调匹配 |
| D6 | 视觉：暖米色底 + 茶绿主 + 陶土强调；禁止紫色渐变、高饱和警示 | 见 `specs/ui-design.md` §0 设计哲学 |

## 三、卡住时的兜底

| 情形 | 怎么办 |
|---|---|
| 不确定 Cocos 3.8 API 行为 | 用 Context7 MCP 查 `cocos creator 3.8` |
| 不确定当前进度 | 读 `specs/PROGRESS.md` + `git log --oneline -20` |
| 不确定某项是否已决策 | 看本文档 §二「已决策」清单 |
| 不确定文件归属 | 看 `.cursorrules` §四「目录约定」与 `system-arch.md` §9「目录最终态」 |
| 用户问的事与已决策冲突 | 明确告诉用户冲突点，征求是否重新评估 |
| 微信 / Native / H5 行为差异 | 看 `system-arch.md` §6.6 平台差异清单 |
| 某步做错了需要回退 | `git reset` 回上一个安全 commit；PROGRESS.md 对应步骤恢复为 `[ ]`；已决策清单不回退 |

## 四、PROGRESS.md 更新模板

> **小任务推进**（改参数、修 bug）：只需更新「当前状态」+ 勾掉 checkbox。
> **里程碑完成 / 新决策 / 架构变更**：执行以下全部 5 步。

每次推进结束后，按以下方式更新 `specs/PROGRESS.md`：

1. **「当前状态」**：覆盖 3 行（当前里程碑 / 正在做 / 下一步）。
2. **「里程碑追踪」**：把刚完成的子任务的 `[ ]` 改成 `[DONE YYYY-MM-DD]`。
3. **「决策记录」**：如有新决策，**追加一行**到表头（最新在上）。
4. **「未解决问题」**：如答到了某个问题，从清单删除；如冒出新问题，追加。
5. **「当前会话上下文摘要」**：覆盖为最新 1 段，给下一次新会话作引子。
