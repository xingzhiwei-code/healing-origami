# Healing Origami · 游戏玩法规范

> 作用域：本文件定义**玩家视角**的所有玩法决策（操作、反馈、节奏、关卡结构、收藏体系）。
> 工程实现 / 模块拆分见 [`specs/system-arch.md`](./system-arch.md)。
> 视觉 Token / 动效 Token 见 [`specs/ui-design.md`](./ui-design.md)。
> 修改本文件必须同步更新另两份规范并在 PR 描述中说明动机。

---

## 0. 一句话定义

> **Healing Origami**：玩家用手指把一张被「揉散」的纸，沿折痕一折一折地修复回原样。每一关修好一张老明信片、一封旧信、一幅童年简笔画。**没有计时、没有连击、不抢节奏**——这是一款「让你慢下来」的折纸修复游戏。

参考品类：*Paper Fold*（Voodoo）的「折叠成型」机制 + 治愈/手帐风叙事外壳。

---

## 1. 设计哲学（玩法侧）

呼应 [`specs/ui-design.md`](./ui-design.md) §0「茶歇时光，纸上修补」：

| 维度 | 我们做 | 我们不做 |
|---|---|---|
| 节奏 | 单关 40 – 90 s，无计时 | 倒计时、连击、限时活动 |
| 难度 | 关卡靠**片段数**与**折叠顺序耦合**递增 | 故意刁难、需要反应速度的操作 |
| 反馈 | 折错回弹 + 轻提示 | 红屏震屏、扣分、失败动画 |
| 数值 | 通关「完成 / 未完成」二值 | 三星评分、最优步数、排行榜 |
| 留存 | 通关收藏到「手帐本」长期展示 | 体力、签到、强提醒 push |
| 商业 | v1 不开商业化；v2 仅做「主题包」一次性内购 | 强制激励视频、关卡门槛弹窗 |

---

## 2. 核心循环（Core Loop）

```
观察散乱片段 → 想象折叠顺序 → 拖拽折叠 → 看到图案逐步成型 → 通关展示 → 收入手帐本
```

单次循环约 60 s，玩家心理曲线：**好奇 → 思考 → 操控 → 啊哈 → 满足**。

---

## 3. 玩家输入模型

### 3.1 唯一输入

单指拖拽。**禁止**多点缩放、长按菜单、双击。

### 3.2 手势 → 折叠角度

- `TOUCH_START` 命中片段 `F` → 进入「折叠会话」。
- `TOUCH_MOVE` 的位移投影到 `F.foldAxis` 法线方向，按系数 `k` 映射到角度 `α ∈ [0°, 180°]`。
- `k` 设计目标：屏幕短边 1/3 长度的拖拽 ≈ 180°，让玩家既能精细控制又不至于反复横拉。
- `TOUCH_END` 时按角度落点决策：

| 释放角度 | 行为 |
|---|---|
| α ≥ 150° | **吸附** 到 180°，提交折叠，按 `correctOrder` 校验 |
| 0° < α < 150° | **回弹** 到 0°，不视为失败，不计步数 |
| α < 0°（反方向） | 视为「展开已折片段」，仅当 `F` 已被折叠时允许 |

### 3.3 触感反馈（微信振动，经 `WXSDK.vibrateShort`）

| 触发时机 | 强度 |
|---|---|
| 拖拽过 30° / 60° / 90° / 120° / 150° | `light` |
| 吸附到 180° | `medium` |
| 折错顺序回弹 | `light` × 1 + 抖动 |
| 通关完成 | `medium` × 2（间隔 200 ms） |

非微信环境（H5 / Web 预览）静默降级为 noop。

### 3.4 顺序错误反馈

当玩家想折的片段被「应该先折的下层片段」挡住时（穿模检测见 `system-arch.md` §2.3 `FoldOrderResolver`），**不惩罚**，而是：

1. 触发回弹动画（`duration-fold` × 0.6）；
2. 顶部弹出 toast：「这一折要先放后面那张哦」；
3. 高亮提示「应该先折」的那个片段，描边色 `clay-300`，持续 `duration-slow` 后淡出。

> 重要：**严禁**出现「失败」「错误」「Try Again」字样，治愈基调严禁惩罚。

## 4. 关卡构成

### 4.1 单关定义

一关 = **一张完整图案** + **N 个折纸片段** + **一条隐式的折叠顺序**。

| 字段 | 玩法语义 |
|---|---|
| 完整图案 | 通关后玩家看到的「修复成果」（明信片 / 信纸 / 童年画） |
| 片段 | 玩家需要折叠的多边形单元 |
| 折叠顺序 | 隐藏在 `correctOrder` 中；玩家通过观察图案 + 试错感知 |

### 4.2 关卡时长目标

- 章节 1：30 – 50 s
- 章节 2：50 – 80 s
- 章节 3：60 – 120 s

超过此区间应通过减少片段数或简化顺序耦合调整。

### 4.3 评价

仅「完成 / 未完成」二值。**不**记录用时、不记录步数、不显示统计。

## 5. 章节与解锁

| 章节 | 主题 | 关卡数 | 片段数 | 解锁条件 | 美术方向 |
|---|---|---|---|---|---|
| 1. 茶歇 | 简单几何 | 5 | 6–10 | 默认 | 几何色块、暖米底 |
| 2. 旧信 | 信纸折法 | 8 | 10–16 | 通过章节 1 | 牛皮纸、钢笔字 |
| 3. 童年 | 不规则图案 | 10 | 16–24 | 通过章节 2 | 蜡笔、儿童画 |
| 4. 远行（v2 预告） | 立体折纸 | 2 (demo) | — | 通过章节 3 | 留作 v2 |

每章通关后解锁**手帐本**对应内页。

---

## 6. 手帐本（Journal）

### 6.1 定位

留存系统，**不是道具系统**。玩家无法在手帐本里强化、合成、交易任何东西。

### 6.2 内容

| 项 | 来源 |
|---|---|
| 已修复明信片 | 章节 1 通关 |
| 已修复信纸 | 章节 2 通关 |
| 已修复童年画 | 章节 3 通关 |
| 折叠过程回放 | 每关首次通关时录制（关键时间戳 + 折叠顺序） |

### 6.3 交互

- 点击某张明信片 → 进入「重温」视图，自动播放当时的折叠回放（`duration-celebrate` × N）。
- 长按 → **暂不开放**（v2 考虑分享给微信好友）。

## 7. 视觉与动效（引用 Design Tokens）

> 所有数值禁止裸写，必须经 `assets/scripts/ui/DesignTokens.ts`（M5 落地）。

| 元素 | Token |
|---|---|
| 全屏背景 | `paper-50` |
| 纸张正面纹理底色 | `paper-100` |
| 纸张背面（折过来看到的） | `paper-200`（比正面深一刻度，呼应「另一面」） |
| 折痕轴线（拖拽中可见） | `clay-300`，alpha 12% |
| 顺序错误高亮描边 | `clay-300` |
| 通关图案高亮 | `sage-500` 描边 + `shadow-paper-lift` |
| 折叠手指控制中缓动 | 线性映射，不加缓动 |
| 折叠自动吸附 | `ease-fold` + `duration-fold` (600 ms) |
| 折叠回弹 | `ease-paper` + `duration-fold × 0.6` |
| 通关展示 | `ease-bounce-soft` + `duration-celebrate` (1200 ms) |
| 顶栏 / 弹窗渐入 | `ease-paper` + `duration-base` (240 ms) |

**单屏同时进行的动画数量上限：3** —— 与 `ui-design.md` §6 一致。

## 8. 失败 / 异常体验

| 场景 | 处理 |
|---|---|
| 玩家折错顺序 | 见 §3.4，不惩罚 |
| 玩家主动反折 | 允许，恢复到 0°，纸片回到「散乱」状态 |
| 玩家退出关卡 | 进度**不保留**（一关被视作一次完整冥想，下次重新开始） |
| 微信网络中断（远程贴图未拉到） | 留在关卡选择页，toast：「图案还在路上，稍后再试」 |
| 微信内存吃紧（`wx.onMemoryWarning`） | 立刻 destroy 未进入的章节贴图缓存，向玩家完全透明 |

## 9. 反例（禁止）

- 倒计时、连击数、星级评分、最优步数排行；
- 「失败」「再试一次」「错误」等字样；
- 强制激励视频解锁关卡；
- 通关后立刻弹「下一关」自动跳转——必须先让玩家看完 1.2 s 展示并主动点击；
- 红色 / 高饱和警示色（与 `ui-design.md` §1.6 语义色约束一致）；
- 任何高频弹窗（push、签到、首充、限时礼包）。
